# Review Rubric

Use this file before claiming the frontend is complete.

## Litmus checks

- Is the product or brand unmistakable in the first screen?
- Is there one dominant visual anchor?
- Can the page or screen be understood by scanning headings and primary actions?
- Does each section or region have one main job?
- Are cards truly necessary?
- Does motion improve hierarchy, affordance, or atmosphere?
- Does the design still feel intentional if decorative shadows are removed?
- Does the layout remain coherent on mobile?
- Do fixed, sticky, or floating elements stay clear of key content?
- Is the copy believable and product-specific?

## Common failure modes

- Generic SaaS card grid as the first impression
- Weak brand or product identity
- Strong headline but no clear action
- Busy background imagery behind text
- Repeated sections that all say the same thing
- Dashboard UI broken into too many framed panels
- Decorative gradients or icons that add noise instead of clarity
- Hero content that overflows the viewport because of persistent chrome
- Placeholder copy that destroys credibility

## Final pass

- Tighten the first screen.
- Remove one unnecessary UI device from each section if possible.
- Shorten copy until every line earns its space.
- Recheck spacing, contrast, and breakpoints.
- Confirm the result still matches the brief that was approved.
